📘 Лекция 4: Графика и мультимедиа

🎯 Цель лекции: Сформировать понимание принципов встраивания графических и мультимедийных элементов в веб-страницы, а также освоить основные HTML-теги и атрибуты для работы с изображениями, аудио и видео.

Форматы изображений

Стандарт HTML не содержит обязательного списка поддерживаемых форматов изображений, поэтому разные пользовательские агенты могут поддерживать разные наборы.

Некоторые распространённые форматы изображений, которые можно использовать в HTML:

JPEG (Joint Photographic Experts Group)

Подходит для фотографий, изображений с большим количеством разноцветных деталей, плавным переходом яркости и контраста. Использует сжатие с потерями, что может ухудшать качество изображения.

PNG (Portable Network Graphics)

Подходит для изображений с прозрачным фоном, иконок, логотипов и графики с чёткими линиями. Поддерживает сжатие без потерь, что обеспечивает высокое качество, но файлы обычно больше, чем в JPEG.

SVG (Scalable Vector Graphics)

Векторный формат, который позволяет масштабировать изображения без потери качества. Подходит для логотипов, иконок, иллюстраций, диаграмм. Можно встраивать напрямую в HTML-код через тег <svg>.

GIF (Graphics Interchange Format)

Используется для простых анимаций, изображений с ограниченной цветовой палитрой и иконок. Ограничен 256 цветами, не подходит для сложных изображений.

WebP

Современный формат, разработанный Google. Обеспечивает эффективное сжатие, поддерживает прозрачность и анимацию. Работает во всех современных браузерах, но может не поддерживаться в старых версиях.

AVIF (AV1 Image File Format)

Обеспечивает высокое качество при минимальном размере файла, но поддерживается не во всех браузерах.

APNG (Animated Portable Network Graphics)

Подходит для lossless-анимации, является альтернативой GIF.

При выборе формата изображения стоит учитывать его назначение, сложность, необходимость прозрачности, а также поддержку браузерами. Часто для оптимизации доставки изображений используют комбинацию разных форматов на одном сайте.

Тег <img>

Изображения в HTML вставляют с помощью тега <img> или с помощью CSS. Выбор метода зависит от задачи и типа изображения.

Подходит для контентных изображений: логотипов, фото, иллюстраций, графиков и других. Тег <img> указывает браузеру, где нужно вставить картинку.

Некоторые атрибуты тега:

  • src — указывает путь до изображения. Путь может быть относительным (если картинка находится внутри папки проекта) или абсолютным (если изображение лежит на другом сервере). Например, если сам index.html лежит в корневой папке проекта, а изображение — в папке images/, то пишут так: <img src="images/photo.jpg" alt="Фото">.
  • alt — добавляет альтернативный текст, который отображается, если изображение не загрузилось. Важно для доступности (помогает скринридерам озвучивать контент) и SEO (поисковики тоже его учитывают).
  • width — задаёт ширину изображения.
  • height — задаёт высоту изображения.

Тег <img> одиночный: он не требует закрывающей пары. В HTML5 можно писать его так, без косой черты на конце: <img src="...">.

Подписи изображений

В HTML для добавления подписей к изображениям используется связка элементов <figure> и <figcaption>. Эти элементы позволяют семантически выделить изображение и его подпись, что улучшает структуру документа и взаимодействие с вспомогательными технологиями.

Основные принципы использования

  1. Тег <figure> используется для группировки изображения и подписи. Он позволяет рассматривать изображение и подпись как единое целое для стилизации и макета.
  2. Тег <figcaption> содержит текст подписи. Он должен быть вложен внутрь <figure> и может располагаться либо первым, либо последним элементом внутри этого контейнера. Если не применять CSS-стили, подпись будет отображаться либо над изображением, либо под ним — в зависимости от того, какой элемент стоит первым.
  3. Альтернативный текст (alt у <img>) задаёт альтернативный текст для изображения, который отображается, когда картинка недоступна или для чтения экранным диктором. figcaption же служит более подробной и контекстной подписью для пользователя.

Пример кода

<figure>
  <img src="mountains.jpg" alt="Горный пейзаж на закате" />
  <figcaption>Вид на горы в Норвегии на закате</figcaption>
</figure>

В этом примере изображение с атрибутом alt сопровождается подписью.

Стилизация

С помощью CSS можно изменить внешний вид подписи и изображения, например:

  • задать рамку и внутренние отступы для контента с помощью border и padding для элемента <figure>;
  • выровнять текст подписи по центру с помощью text-align;
  • изменить цвет фона с помощью background-color для <figcaption>.

Пример стилей:

figure {
  border: 1px solid #245488;
  padding: 10px;
  min-width: 250px;
}

figcaption {
  text-align: center;
  background-color: #bac8d6;
}

img {
  width: 100%;
}

Дополнительные возможности

  • Размещение подписи сбоку. С помощью свойства float можно расположить подпись справа или слева от изображения, но это может повлиять на расположение всех нижележащих элементов.
  • Составная подпись. Внутри <figcaption> можно использовать различные элементы строчного или блочного уровня, которые логически относятся к подписи.
  • Несколько изображений с подписями. С помощью этих элементов можно добавлять подписи к нескольким изображениям.

Таким образом, комбинация <figure> и <figcaption> обеспечивает семантическую связь между изображением и его описанием, что улучшает структуру документа и взаимодействие с пользователями.

Аудио и видео

В HTML для встраивания аудио- и видеоконтента используются специальные теги, которые позволяют интегрировать мультимедиа прямо в документ без сторонних плагинов.

Тег <audio>

Используется для встраивания аудиофайлов (музыки, звуковых эффектов, аудиокниг и т. д.).

Базовый синтаксис:

<audio src="путь-к-аудиофайлу.mp3" controls></audio>

Атрибут src указывает путь к аудиофайлу, а controls добавляет элементы управления (кнопки воспроизведения, паузы, регулировки громкости).

Некоторые атрибуты тега <audio>:

  • autoplay — автоматическое воспроизведение при загрузке страницы (работает не во всех браузерах без muted);
  • loop — циклическое воспроизведение трека;
  • muted — установка беззвучного состояния;
  • preload — определение способа загрузки файла (значения: none, metadata, auto).

Для обеспечения совместимости с разными браузерами рекомендуется использовать несколько источников через вложенные элементы <source>. Браузер выберет первый поддерживаемый формат.

Примеры форматов аудио: MP3, WAV, Ogg.

Тег <video>

Предназначен для вставки видеоконтента.

Базовый синтаксис:

<video src="путь-к-видео.mp4" controls></video>

Атрибут src указывает путь к видеофайлу, controls — элементы управления.

Некоторые атрибуты тега <video>:

  • autoplay — автовоспроизведение;
  • loop — зацикливание видео;
  • muted — отключение звука;
  • preload — определение способа загрузки (значения: auto, metadata, none);
  • poster — указание изображения, которое будет отображаться до начала воспроизведения;
  • width и height — задание размеров видеоплеера.

Для обеспечения кроссбраузерной совместимости также рекомендуется использовать вложенные элементы <source>. Браузер выберет первый поддерживаемый формат.

Основные форматы видео: MP4 (H.264), WebM, Ogg.

Дополнительные возможности

  • Субтитры. Для их добавления используется элемент <track> внутри <video>. Указывается язык и файл с текстом (формат .vtt).
  • Стилизация. Элементы управления видео и аудио можно стилизовать через CSS.
  • JavaScript. С помощью API можно управлять аудио и видео, например, создавать собственные элементы управления.
Важно учитывать: современные браузеры часто блокируют автовоспроизведение медиа с звуком, поэтому для надёжного использования autoplay рекомендуется комбинировать его с muted или обеспечивать воспроизведение только после взаимодействия пользователя со страницей.
