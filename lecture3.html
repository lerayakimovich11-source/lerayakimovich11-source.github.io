📘 Лекция 3: Гиперссылки и пути к файлам

🎯 Цель лекции: Сформировать понимание принципов организации навигации в веб-пространстве с помощью гиперссылок, а также освоить правила построения абсолютных и относительных путей к файлам для связывания страниц и ресурсов внутри веб-сайта.

Тег <a>

Тег <a> (от англ. anchor — якорь) в HTML — это элемент, который используется для создания гиперссылок. Он позволяет направлять пользователя на другие веб-страницы, файлы, определённые места внутри той же страницы, адреса электронной почты, номера телефонов или выполнять другие действия (например, запускать JavaScript).

Базовый синтаксис

У тега <a> есть открывающий и закрывающий теги: <a> и </a>. Между ними помещается содержимое ссылки — обычно текст, но это может быть и изображение, иконка, сложная разметка.

Пример:

<a href="https://example.com">Перейти на сайт Example</a>

В браузере такая ссылка будет выглядеть как подчёркнутый синий текст.

Обязательный атрибут href — ключевой атрибут тега <a>. Он задаёт адрес ресурса, на который ведёт ссылка. Без этого атрибута тег создаёт лишь заполнитель, а не настоящую ссылку.

В значении href можно использовать:

  • абсолютный URL (с протоколом и доменом);
  • относительный путь (относительно текущей страницы);
  • якорь (переход к элементу на той же странице, для этого перед ID элемента ставится символ #);
  • псевдопротоколы (mailto, tel и другие).

Некоторые атрибуты тега <a>

  • target — указывает, в каком окне или вкладке откроется ссылка. Некоторые значения:
    • _self — по умолчанию, открытие в текущей вкладке/окне;
    • _blank — открытие в новой вкладке/окне;
    • _parent, _top — используются во фреймах;
    • имя окна/фрейма — для сложных случаев навигации.
  • rel — определяет отношение между текущим документом и связанным ресурсом. Например, nofollow указывает поисковым системам не учитывать эту ссылку при ранжировании.
  • download — указывает, что при клике на ссылку должен загрузиться указанный ресурс.
  • hreflang — определяет язык связанного документа.
  • type — указывает MIME-тип связанного документа.

Также тег <a> поддерживает глобальные атрибуты HTML и атрибуты событий.

Дополнительные возможности

  • Якорные ссылки. Позволяют переходить к определённым разделам внутри одной страницы.
  • Ссылки на локальные файлы (с использованием file:///).
  • Ссылки, которые при клике открывают почтовый клиент (с использованием mailto:).
  • Ссылки на номера телефонов (с использованием tel:).
  • Запуск JavaScript-кода при использовании javascript: в значении href.
Важно: если связанная страница будет удалена или перемещена, ссылка перестанет работать.

Атрибут title

Атрибут title в HTML — это глобальный атрибут, который позволяет добавить дополнительную текстовую информацию об элементе. В графическом браузере её чаще всего можно увидеть в виде всплывающей подсказки при наведении курсора мыши на элемент.

Синтаксис

<element title="текст всплывающей подсказки">

Строка с текстом подсказки должна быть заключена в двойные или одинарные кавычки.

Где можно использовать

Атрибут title можно добавлять к большинству HTML-элементов, например:

  • <a> (ссылки);
  • <button>, <input>, <textarea> (элементы форм);
  • <img> (изображения);
  • <abbr> (аббревиатуры);
  • <div>, <p> и другим.

Пример с ссылкой:

<a href="https://example.com" title="Перейти на сайт Example">Официальный сайт</a>

При наведении курсора на текст «Официальный сайт» появится системная подсказка с текстом «Перейти на сайт Example».

Пример с изображением:

<img src="avatar.jpg" alt="Фотография пользователя" title="Нажмите, чтобы открыть профиль" />

Особенности

  • Вид подсказки зависит от браузера и настроек операционной системы. Изменить его с помощью HTML-кода или стилей нельзя.
  • Атрибут принимает только текст. Если внутри title написать HTML-теги, браузер отобразит их как обычный текст, а не как разметку.
  • Всплывающие подсказки, созданные с помощью title, могут быть недоступны при использовании смартфонов или других устройств с ограниченными возможностями просмотра.
  • Поисковые системы могут использовать содержимое title как дополнительный сигнал, но оно не должно рассматриваться как основной контент.
  • Скринридеры могут использовать title как дополнительную подсказку.

Ограничения

  • Не подходит для кастомного внешнего вида подсказки (кастомизация цветов, отступов, стрелочек и т. д.). Для этого потребуется реализовывать собственный механизм всплывающих подсказок на JavaScript и CSS.
  • Рекомендуется использовать атрибут умеренно, чтобы не перегружать пользователя избыточной информацией.

Если требуется более сложная реализация подсказок (например, с использованием изображений или форм ввода), можно использовать специальные JavaScript-библиотеки или плагины.

Навигация по странице, использование тега id

Тег id позволяет создавать якоря (anchor points) на веб-странице, к которым можно осуществлять плавную или мгновенную навигацию.

Как это работает

1. Создание якоря (целевого элемента)

<!-- Любой элемент с уникальным id -->
<h2 id="section1">Раздел 1</h2>
<div id="contacts">Контактная информация</div>
<p id="important-text">Важный текст</p>

2. Создание ссылки на якорь

<!-- Ссылка на элемент с id="section1" -->
<a href="#section1">Перейти к разделу 1</a>

<!-- Ссылка на текущей странице -->
<a href="#contacts">Контакты</a>

<!-- Ссылка с другой страницы -->
<a href="page.html#section3">Раздел 3 на другой странице</a>

Важные правила использования id

  • Уникальность — id должен быть уникальным в пределах одной HTML-страницы;
  • Формат — начинается с буквы (a-z, A-Z), может содержать цифры, дефисы, подчеркивания, двоеточия, точки;
  • Регистрозависимость — id="about" и id="About" — разные идентификаторы;
  • Пробелы запрещены — в значении id не должно быть пробелов.

Ссылки на email и телефон в HTML

mailto и tel — это специальные схемы URL в HTML, которые позволяют создавать кликабельные ссылки на электронную почту и телефонные номера.

Схема mailto

Схема mailto используется для создания ссылок, которые открывают почтовый клиент пользователя с уже заполненными полями адресата, темы и тела письма.

Синтаксис:

<a href="mailto:email@example.com">Отправить email</a>

Дополнительные возможности:

  • Можно указать несколько адресов, разделив их запятыми: mailto:email1@example.com,email2@example.com.
  • Для предзаполнения темы письма используется параметр subject, для тела сообщения — body. Пробелы и спецсимволы в этих параметрах нужно кодировать (URL-кодировать).
  • Параметры cc (копия) и bcc (скрытая копия) позволяют указать дополнительные адреса получателей.
При клике на такую ссылку откроется почтовая программа пользователя с уже заполненными полями. Если почтовая программа по умолчанию не установлена, появится окно выбора программы.

Схема tel

Схема tel используется для создания ссылок на телефонные номера. При клике по такой ссылке на смартфоне открывается приложение для звонков с набранным номером. На компьютере это может работать, если установлена программа для звонков.

Синтаксис:

<a href="tel:+79001111111">+7 900 111-11-11</a>

Важные нюансы:

  • Номер должен быть указан в международном формате — с + и кодом страны.
  • В атрибуте href рекомендуется убирать пробелы и дефисы, чтобы избежать проблем с распознаванием.
  • Поведение ссылки может зависеть от устройства, операционной системы и браузера. Например, в Chrome, Edge, Firefox на Windows может появиться предложение использовать программу для звонка.
