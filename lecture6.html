📘 Лекция 6: Формы

🎯 Цель лекции: Сформировать понимание принципов создания и обработки веб-форм в HTML, а также освоить основные типы полей ввода, атрибуты и семантическую структуру форм для сбора и отправки пользовательских данных.

Зачем нужны формы

Формы в HTML — это основной инструмент для сбора и отправки данных от пользователя на сервер. Без них большинство современных сайтов было бы просто «витриной», с которой нельзя взаимодействовать.

Вот 4 ключевые причины, зачем нужны формы:

1. Взаимодействие с пользователем (обратная связь)

Формы позволяют пользователю что-то передать сайту. Это основа любого интерактива:

  • Поиск: строка поиска на Яндексе или Google (поле ввода + кнопка).
  • Авторизация: поля «Логин» и «Пароль», чтобы войти в личный кабинет.
  • Регистрация: создание нового аккаунта.
  • Обратная связь: форма «Напишите нам», «Заказать звонок» или «Оставить отзыв».

2. Сбор данных и ввод информации

Формы помогают структурировать ввод данных там, где простого текста недостаточно:

  • Комментарии под статьей или постом в соцсети.
  • Онлайн-заказы: указать адрес, количество товара, способ доставки.
  • Заполнение анкет (опросы, тесты, голосования).
  • Загрузка файлов (прикрепить фото к объявлению или резюме).

3. Управление контентом (для владельцев сайтов)

Через формы работают панели администратора:

  • Добавление новой статьи в блог (заголовок, текст, теги).
  • Редактирование товаров в интернет-магазине (цена, описание, остаток на складе).

4. Техническая необходимость: передача данных на сервер

С точки зрения кода, форма (<form>) решает задачу, которую нельзя сделать просто ссылкой или кнопкой:

  • Она группирует поля ввода (<input>, <textarea>, <select>).
  • Определяет куда отправить данные (атрибут action).
  • Определяет как отправить (method="GET" или POST). Например, пароли нельзя передавать через GET — только через POST, чтобы они не светились в адресной строке.

Тег <form> в HTML

Тег <form> в HTML используется для создания форм, которые позволяют пользователям вводить данные и отправлять их на сервер. Он служит контейнером для элементов управления, таких как поля ввода, кнопки и другие элементы.

Основные атрибуты тега <form>

  • action — указывает URL-адрес, куда будут отправлены данные формы. Это может быть полный URL или относительный путь. Если атрибут не указан, данные отправятся на ту же страницу, где находится форма.
  • method — определяет метод отправки данных. Возможны два основных значения:
    • GET — данные добавляются к URL в виде строки запроса (пары «параметр=значение»). Подходит для поиска и фильтров, но имеет ограничения по объёму данных и видимости в адресной строке.
    • POST — данные передаются в теле HTTP-запроса, не отображаясь в URL. Используется для форм с конфиденциальной информацией, загрузки файлов и других случаев, где важен объём данных.
  • enctype — указывает тип кодирования данных формы при отправке на сервер (только для метода POST). Возможные значения:
    • application/x-www-form-urlencoded — значение по умолчанию, если атрибут не задан.
    • multipart/form-data — используется, если в форме есть элемент <input type="file">.
  • autocomplete — управляет автозаполнением полей браузером. Может принимать значения on (включено) или off (выключено).
  • novalidate — отключает встроенную валидацию формы браузером. Используется, если данные нужно проверять с помощью собственного скрипта.
  • target — определяет, где открыть результат после отправки формы (например, в том же окне _self или в новом _blank).
  • accept-charset — задаёт кодировку, в которой сервер принимает данные из формы. Самая распространённая кодировка — UTF-8.

Также тег <form> поддерживает глобальные атрибуты HTML и атрибуты, связанные с событиями.

Дополнительные особенности

  • Каждый элемент внутри формы должен иметь атрибут name, который служит идентификатором при отправке данных на сервер.
  • Для форм с загрузкой файлов необходимо одновременно установить method="post" и enctype="multipart/form-data", а также включить элемент <input type="file">.
  • Метод GET не подходит для отправки конфиденциальных данных, так как они могут быть видны в URL.

Пример использования атрибутов:

<form action="/submit" method="post" enctype="multipart/form-data" target="_blank" novalidate autocomplete="off">
    <label for="name">Ваше имя:</label>
    <input type="text" id="name" name="name" required>
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>
    <button type="submit">Отправить</button>
</form>

Основные типы полей ввода в HTML

1. text — текстовое поле

Самое простое поле для ввода одной строки текста (имя, город, название).

<input type="text" placeholder="Введите имя">

2. password — пароль

Скрывает вводимые символы (заменяет их точками или звездочками).

<input type="password" placeholder="Введите пароль">

3. email — электронная почта

Проверяет, есть ли символ @ и точка в домене. На мобильных устройствах показывает удобную клавиатуру с @ и .com.

<input type="email" placeholder="user@example.com">

4. number — число

Ограничивает ввод только цифрами. Позволяет использовать стрелки вверх/вниз для изменения значения. Можно задать минимум, максимум и шаг.

<input type="number" min="1" max="100" step="1">

5. checkbox — флажок (галочка)

Позволяет выбрать один или несколько вариантов из списка. Возвращает значение on, если отмечен (или заданное value).

<input type="checkbox" name="agree"> Согласен с условиями

Кнопки в HTML-формах

Кнопки в HTML-формах нужны для отправки данных, очистки или выполнения пользовательских действий. Рассмотрим основные типы.

1. submit — кнопка отправки формы

Самая важная кнопка. При нажатии отправляет все данные формы на сервер (по адресу из action).

<form action="/send" method="post">
    <input type="text" name="username">
    <button type="submit">Отправить</button>
</form>

Особенности:

  • Отправляет форму (вызывает событие submit)
  • Можно использовать как <button type="submit">, так и <input type="submit">

2. reset — кнопка сброса

Очищает все поля формы до первоначальных значений.

<form>
    <input type="text" value="Текст">
    <input type="reset" value="Очистить всё">
</form>

Важно: используется редко, т.к. пользователь может случайно стереть все данные.

3. button — обычная кнопка

Ничего не делает сама по себе. Нужна для выполнения JavaScript-кода (открыть окно, проверить данные, показать подсказку).

<button type="button" onclick="alert('Привет!')">Нажми меня</button>

Многострочное поле ввода в HTML — <textarea>

В отличие от <input> (который предназначен только для одной строки), <textarea> позволяет пользователю вводить несколько строк текста.

Базовый пример

<textarea rows="4" cols="50">
Текст по умолчанию (можно сразу написать)
</textarea>

Основные атрибуты

АтрибутЗначениеЧто делает

colsчисло (например, 40)Ширина поля в символах

Примеры использования

1. Простое поле для комментария

<textarea name="comment" placeholder="Напишите ваш комментарий..." rows="4" cols="50"></textarea>

2. Поле с ограничением длины (например, отзыв)

<textarea name="review" maxlength="500" minlength="10" rows="5" placeholder="Ваш отзыв (от 10 до 500 символов)"></textarea>

3. Только для чтения (лицензионное соглашение)

<textarea readonly rows="10">
Здесь длинный текст лицензионного соглашения...
Пользователь может пролистывать, но не может редактировать.
</textarea>

Группировка полей — теги <fieldset> и <legend>

Они нужны, чтобы объединять связанные поля формы в визуальные блоки. <fieldset> создает рамку, а <legend> — заголовок внутри этой рамки.

Простой пример:

<fieldset>
    <legend>Контактные данные</legend>
    <label>Имя: <input type="text" name="name"></label>
    <label>Email: <input type="email" name="email"></label>
</fieldset>

Где это полезно:

  • Длинные формы (регистрация, оформление заказа)
  • Анкеты и опросы
  • Платежные формы (данные карты, адрес доставки)

Дополнительные возможности: Если добавить атрибут disabled в <fieldset>, отключатся сразу все поля внутри. Также можно вкладывать группы друг в друга.

Главный плюс: пользователю легче ориентироваться, а скринридеры правильно озвучивают заголовок группы перед полями.

Короткое правило: используйте группировку, когда в форме больше 4-5 полей или есть разные смысловые разделы (личные данные, оплата, соглашения).

Выпадающие списки в HTML — тег <select>

Выпадающий список позволяет пользователю выбрать один или несколько вариантов из предложенного набора, экономя место на странице.

Основная конструкция

<select name="city">
    <option>Москва</option>
    <option>Санкт-Петербург</option>
    <option>Казань</option>
    <option>Новосибирск</option>
</select>

Основные атрибуты <select>

  • multiple — позволяет выбрать несколько вариантов (обычно с зажатым Ctrl или Cmd).
  • size — задает видимое количество пунктов (вместо выпадающего списка появляется прокручиваемая область).
  • required — делает список обязательным для выбора.

Атрибуты для <option>

  • selected — задает вариант по умолчанию.
  • value — задает значение, которое отправится на сервер (если не указать, отправится текст опции).
  • disabled — отключает отдельный пункт (нельзя выбрать).

Группировка внутри списка — <optgroup>

<select name="car">
    <optgroup label="Немецкие">
        <option>BMW</option>
        <option>Mercedes</option>
        <option>Audi</option>
    </optgroup>
    <optgroup label="Японские">
        <option>Toyota</option>
        <option>Honda</option>
        <option>Nissan</option>
    </optgroup>
</select>

Полный пример формы

<form action="/order" method="post">
    <label>Город доставки:
        <select name="city" required>
            <option value="">--- Выберите город ---</option>
            <option>Москва</option>
            <option>Санкт-Петербург</option>
            <option>Казань</option>
        </select>
    </label>
    <label>Способ оплаты:
        <select name="payment">
            <option>Картой при получении</option>
            <option>Наличными</option>
            <option selected>Онлайн картой</option>
        </select>
    </label>
    <label>Дополнительные услуги (можно выбрать несколько):
        <select name="services" multiple size="3">
            <option>Страховка</option>
            <option>Срочная доставка</option>
            <option>Подарочная упаковка</option>
        </select>
    </label>
    <button type="submit">Оформить</button>
</form>

Когда использовать выпадающие списки?

Подходит для:

  • Выбора страны, города, языка
  • Категорий товаров
  • Способов оплаты и доставки
  • Любого списка с 4-15 вариантами

Не подходит для:

  • Менее 3 вариантов (лучше использовать радиокнопки)
  • Более 15 вариантов (нужен поиск или автодополнение)

Главное отличие от радиокнопок: Выпадающий список экономит место — показывает только выбранный вариант, а все остальные скрыты. Радиокнопки показывают все варианты сразу. Для 10+ вариантов всегда используйте <select>.

Поля выбора даты и цвета в HTML

Это современные типы полей <input>, которые предоставляют удобные визуальные инструменты для выбора.

Выбор даты — type="date"

Открывает календарик для выбора дня, месяца и года.

<input type="date" name="birthday">

Дополнительные атрибуты:

<input type="date" min="2024-01-01" max="2024-12-31" value="2024-06-15">

Формат отправки на сервер: всегда ГГГГ-ММ-ДД (2024-12-31), независимо от того, как это выглядит в браузере пользователя.

Другие типы дат и времени

ТипЧто выбираетПример
datetime-localДата + время (без часового пояса)2024-12-31T15:30
monthМесяц и год2024-12
weekНеделя и год2024-W52
timeТолько время15:30

Выбор цвета — type="color"

Открывает палитру (пипетку) для выбора цвета.

<input type="color" name="bg-color" value="#3498db">

Формат значения: шестнадцатеричный код #RRGGBB (например, #ff0000 для красного).

Важные нюансы: Все эти поля имеют отличную браузерную поддержку, но внешний вид отличается в разных браузерах (в Chrome один календарь, в Safari — другой). Если нужен единый стиль на всех устройствах, используют JavaScript-библиотеки.

Простой пример формы:

<form>
    <label>Дата рождения: <input type="date" name="dob" required></label><br>
    <label>Время встречи: <input type="time" name="meeting-time"></label><br>
    <label>Любимый цвет: <input type="color" name="fav-color" value="#ff0000"></label><br>
    <button type="submit">Отправить</button>
</form>

Главный плюс: не нужно писать сложный JavaScript для календаря или палитры — браузер делает всё сам.

Связка текста и поля в HTML-формах

Главный инструмент для связки — тег <label>. Он привязывает текстовую подпись к полю ввода.

Два способа связки

Способ 1 — обёртка (самый удобный):

<label>Имя: <input type="text" name="name"></label>

Способ 2 — через атрибут for (нужен id у поля):

<label for="user-email">Email:</label>
<input type="email" id="user-email" name="email">

Зачем это нужно?

  • Кликабельность: при нажатии на текст курсор автоматически попадает в поле (или отмечает чекбокс/радиокнопку).
  • Доступность: скринридеры озвучивают текст подписи вместе с полем.

Главное правило: У каждого поля ввода должен быть свой <label>. Не используйте просто текст рядом с полем — оборачивайте его в <label>.

Особенности для чекбоксов и радиокнопок

Здесь связка особенно важна, так как маленькую кнопку трудно попасть мышкой.

<label>
    <input type="checkbox" name="agree"> Я согласен с условиями
</label>

<label>
    <input type="radio" name="gender" value="male"> Мужской
</label>

Короткий итог

  • <label> = текст + поле в одном флаконе
  • Можно обернуть поле в <label> или использовать for с id
  • Результат: удобнее нажимать, лучше для экранных дикторов
